GET - запрос контента с сервера

http://www.mysite.com/images?pic1=100&pic2=250

www.mysite.com - имя хоста
/images - путь запроса
? - разделитель обозначающий, что далее следуют параметра запроса

параметры перечисляются в виде ключ=значение
если их несколько то разделяются &

POST - публикация информации на сервере

POST /images/add/json HTTP/1.1
Accept: application/json
Content-Type: application/json
Content-Length: 32
Host: mysite.com

{
"Id" : 123,
"Image" : "Pic1.jpeg"
}

Response Status Codes

- 1xx - информационный. запрос получен сервер готов к продолжению
- 2хх - успешный. Запрос получен, понытен и обработан
- 3хх - перенаправление. Для обработки запроса нужно выполнить следующие действия
- 4хх - ошибка клиента. Запрос содержит ошибки или не соответствует протоколу
- 5хх - ошибки сервера. Сервер не смог обработать запрос, хотя запрос был составлен верно


200 OK - запрос получен и успешно обработан
201 Create - запрос получен и успешно обработан, в результате чего создан новый ресурс или его экземпляр
301 Moved Permanently - запрашиваемый ресурс был перемещен (навсегда) и последующие запросы к нему долдны происходить по новому адресу
307 Temporary Redirect - перемещение временное
400 Bad request - "плохой запрос"
403 forbidden - запрос понятен, но требуется авторизация (отказано за недостаточностью прав доступа)
404 Not Found - сервер не нашел ресурс по данному адресу
500 Internal Server Error - "Упс Что-то пошло не так"
501 Not implemented - сервер не поддерживает функциональность для ответа на этот запрос
505 HTTP Version Not Supported - сервер не поддерживает указанную версию HTTP-протокола


HTTPS - полностью синтаксически идентичен HTTP - + дополнительное шифрование и порт по умолчанию (443)

Шифруется между HTTP и ТСP (то есть между прикладным и транспортным уровнем)

Стандарт шифрования - TLS

Шифрование происходит перед тем как информация попадает на транспортный уровень
